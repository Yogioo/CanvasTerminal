## Command

```cmd
msdf-atlas-gen.exe ^
  -font "C:/Windows/Fonts/NotoSansSC-VF.ttf" ^
  -type mtsdf ^
  -format png ^
  -json assets/fonts/msdf/atlas.json ^
  -imageout assets/fonts/msdf/atlas.png ^
  -size 48 ^
  -pxrange 4 ^
  -charset assets/fonts/msdf/charset.txt ^
  -fontname "NotoSansSC-VF"
```

## Charset

Contains **3896 code points** (95 ASCII printable + 3801 non-ASCII). The atlas contains **3893 glyphs** (3 code points absent from Noto Sans SC):

- ASCII printable characters (U+0020–U+007E)
- GB2312 Level 1 Chinese characters (区16-55, 3755 chars)
- All unique non-ASCII characters from `src/**/*.rs` (623 chars, mostly subset of GB2312)
- Additional common CJK punctuation and symbols (14 chars): ？；『』【】《》…←￥✔○■□▲▼★☆

### Characters NOT in Noto Sans SC (3 chars, will show tofu):
- U+23F8 ⏸ (pause symbol)
- U+2714 ✔ (check mark alternative)
- U+2715 ✕ (cross mark) — these are used in source code but Noto Sans SC does not include them.
  - ✓ (U+2713) IS available in the font.

### Characters excluded (emoji, not in font):
- 🍅 💾 📊 📋 📝 — these are emoji-only and rendered via egui text fallback.

## Regeneration

To regenerate the charset spec:

```bash
python scripts/gen_msdf_charset.py
```

This generates GB2312 Level 1 (~3755 common Chinese characters), collects all unique non-ASCII characters from `src/**/*.rs`, merges with common CJK punctuation and symbols, and writes `assets/fonts/msdf/charset.txt`.

## Coverage Strategy

- **GB2312 Level 1 + project chars** approach: 3893 glyphs in atlas covering ~3755 common Chinese characters plus project-specific symbols.
- **Tofu fallback**: characters not in the atlas (emoji, rare/obscure glyphs) display as tofu (missing glyph box). GB2312 Level 1 covers ~99% of common Chinese text, so most user-typed Chinese titles should display.
- **Future**: later implement dynamic atlas or cosmic-text shaping for full fallback support.

## Notes

- Texture format: `Rgba8Unorm` (NOT sRGB – MSDF data is linear).
- GPU rendering via egui-wgpu PaintCallback with WGSL MSDF shader.
- y-origin = bottom (default). UV Y flipped for wgpu texture coordinates.
