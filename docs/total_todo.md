## 重要
当前最最重要的事情是跑通编程循环, 因为有了编程循环, 那么就可以自动化迭代了.
1. 用户输入需求
2. 用户与AI'CEO'讨论需求详情, 判断需求是否合理 (一个Terminal+Pi即可实现), 确定后将需求拆解成多个可执行任务生成 artifacts/{feature}_Tasks.md
3. 'Planner'从 artifacts/Tasks.md 中, 选择一个优先级最高的任务, 生成任务详情规划, 技术落地方案 artifacts/{feature}_TODO.md,还有可用AI验收的验收方案 artifacts/{feature}_Tests.md, 如果任务全部完成了, 那么 git add and commit 需要的内容 并且添加上 feature/fixed 等标签
4. 'Executer' 按照 artifacts/{feature}_TODO.md 执行代码编码任务, 并保证编译通过
5. 'Tester' 按照 artifacts/{feature}_Tests.md 进行测试,确认测试是否成功, 如果失败发送相关信息给 'Executer' 要求修复. 如果成功 修改 artifacts/{feature}_Tasks.md 标记完成任务,然后返回'Planner' 让他加继续分配下一个任务
